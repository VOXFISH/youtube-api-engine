# Engine/Core 요구·기능·메트릭 명세서

> Project Rice / OpenGL 기반 유튜브 인터랙티브 게임 엔진  
> 대상 폴더: `Engine/Core/`

---

## 1. 폴더 목적

`Engine/Core`의 역할은 어플리케이션의 시작, 동작 순서, 종료를 관리하는 핵심 제어 기능이다.
Core는 어플리케이션의 시작/종료, 게임 루프, 생명주기 호출, 시간 관리, 로그 기능을 총괄한다.
---

### 4.2 PlayerLoop 관리

| 기능 ID | 기능명 | 설명 |
|---|---|---|
| CORE-LOOP-001 | 프레임 시작 처리 | 프레임 카운트 증가 및 시간 측정을 시작한다. |
| CORE-LOOP-002 | 입력 단계 호출 | `Engine/Input`의 입력 갱신 함수를 호출한다. |
| CORE-LOOP-003 | 업데이트 단계 호출 | 현재 씬 또는 게임 상태 업데이트를 호출한다. |
| CORE-LOOP-004 | 렌더링 단계 호출 | `Engine/Renderer`의 렌더링 진입점을 호출한다. |
| CORE-LOOP-005 | 프레임 종료 처리 | 프레임 시간 기록, 로그, 종료 여부를 판단한다. |

### 4.3 호출 순서 관리

Core에서는 어플리케이션의 시작, 종료, 각 업데이트 단계에서의 호출 순서를 보장한다.
Unity에서는 대략 다음과 같이 이뤄진다.
```txt
Initialize
-> Scene Load
-> Component Awake
-> Component OnEnable
-> Component Start
-> Game Loop
    - FixedUpdate : fixedDeltaTime 기반이므로 프레임 당 여러 번 실행될 수 있음
    - Input
    - Update : DeltaTime 기반 업데이트
    - Render
    - LateUpdate : 카메라 렌더링 이후 일어나는 일에 대한 계산
-> Object OnDisable
-> Object OnDestroy
-> Engine Shutdown
```

| 기능 ID | 기능명 | 설명 |
|---|---|---|
| CORE-LIFE-001 | 엔진 초기화 단계 | Core, Platform, Renderer, Scene 등 하위 시스템을 순서대로 초기화한다. |
| CORE-LIFE-002 | 객체 초기화 호출 | 씬 로드 후 객체와 컴포넌트 초기화 요청을 수행한다. |
| CORE-LIFE-003 | 시작 단계 호출 | 첫 프레임 이전에 Start 계열 함수를 호출한다. |
| CORE-LIFE-004 | 프레임 갱신 호출 | 매 프레임 Update 계열 함수를 호출한다. |
| CORE-LIFE-005 | 종료 단계 호출 | 종료 시 리소스 해제 및 Shutdown 계열 함수를 호출한다. |

### 4.4 시간 관리
이 폴더에서는 게임 엔진의 시간 관리를 총괄한다.
게임 엔진에서는 프레임 시간과 물리 시간을 분리하여 관리하는데, 이는 프레임 시간이 컴퓨터의 성능 및 프로그램의 실행 시간에 따라 유동적이기 때문이다.
예를 들면, 물리 연산은 프레임에 따라 충돌이나 속도가 달라질 수 있으므로, 물리 시간 기준인 FixedUpdate에서 계산해야 한다.
accumulator는 각 프레임마다 몇 변의 fixedUpdate가 이루어지는지 계산하여 실제 시간을 기준으로 DeltaTime을 계산한다.
Unity에서는 Time.DeltaTime과 Time.FixedDeltaTime으로 나타낸다.
또, Time.timeScale 프로퍼티를 통해 시간이 흐르는 속도를 조절할 수 있는데, 이 기능은 많은 프로퍼티에서 사용하므로 Core의 값을 전역값으로 사용한다.

| 기능 ID | 기능명 | 설명 |
|---|---|---|
| CORE-TIME-001 | DeltaTime 계산 | 이전 프레임과 현재 프레임 사이의 시간 차이를 계산한다. |
| CORE-TIME-002 | fixedDeltaTime 계산 | 애플리케이션 시작 이후 누적 실행 시간을 제공한다. |
| CORE-TIME-003 | FrameCount 관리 | 현재까지 실행된 프레임 수를 기록한다. |
| CORE-TIME-004 | FPS 계산 | 일정 구간의 평균 FPS를 계산한다. |
| CORE-TIME-005 | 프레임 시간 기록 | 각 프레임의 소요 시간을 로그 또는 프로파일링 데이터로 저장한다. |
| CORE-TIME-006 | 시간 속도 조절 | 시간에 따라 객체가 움직이는 속도를 제어한다. |

### 4.5 로그 및 진단

| 기능 ID | 기능명 | 설명 |
|---|---|---|
| CORE-LOG-001 | 기본 로그 출력 | Info, Warning, Error 수준의 로그를 출력한다. |

## 7. 권장 산출물

`Engine/Core` 개발 단계에서 다음 산출물을 함께 관리한다.

| 산출물 | 설명 |
|---|---|
| Core 설계 문서 | Application, Time, Logger, Loop 구조 설명 |
| 루프 시퀀스 다이어그램 | Input -> Update -> Render 흐름 |
| 생명주기 다이어그램 | Init -> Start -> Update -> Shutdown 흐름 |
| 성능 로그 샘플 | FPS, DeltaTime, FrameTime 기록 예시 |
| 테스트 로그 | 실행 안정성, DeltaTime, 루프 순서 테스트 결과 |
| Core API 문서 | Application, Time, Logger의 공개 인터페이스 |

---

## 8. 구현 시 고려사항

1. `Engine/Core`는 다른 하위 시스템을 직접 깊게 의존하기보다, 인터페이스를 통해 호출하는 구조가 좋다.
2. `Application`은 엔진의 중심이지만, Renderer, Scene, Input의 세부 구현을 직접 처리하지 않아야 한다.
3. 게임 루프의 단계 순서는 로그로 검증 가능해야 한다.
4. `DeltaTime`은 프레임마다 반드시 갱신되어야 하며, 테스트에서 누적 오차를 측정할 수 있어야 한다.
5. 로그 시스템은 개발 중에는 상세하게, 릴리즈 빌드에서는 최소 오버헤드로 동작해야 한다.
6. 장시간 실행 테스트를 위해 Core 레벨에서 프레임 시간과 메모리 사용량을 주기적으로 기록하는 것이 좋다.

---

## 9. 요약

`Engine/Core`의 핵심 목표는 다음과 같다.

- 엔진 애플리케이션을 안정적으로 시작하고 종료한다.
- 매 프레임 `Input -> Update -> Render` 순서를 보장한다.
- 객체와 컴포넌트의 생명주기 호출 흐름을 관리한다.
- `DeltaTime`, `FrameCount`, `FPS` 등 시간 정보를 정확하게 제공한다.
- 실행 중 오류와 성능 정보를 로그로 남긴다.
- 장시간 실행 중에도 크래시 없이 안정적으로 동작한다.

따라서 `Engine/Core`의 성공 여부는 단순히 “실행된다”가 아니라,  
**정해진 순서로 반복 실행되는가, 시간 계산이 정확한가, 장시간 안정적인가, 진단 가능한 로그가 남는가**로 평가해야 한다.
